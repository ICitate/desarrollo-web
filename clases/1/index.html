1. Esquema cliente-servidor

Los navegadores web se comunican con los servidores web usando HTTP (Hyper Text Transfer Protocol) en Espa√±ol (Protocolo de transferencia de hipertexto). HTTP es un protocolo para buscar recursos como por ejemplo, documentos HTML. Es la base de cualquier intercambio de datos en la Web y sigue un esquema de tipo cliente-servidor. Lo que significa que las solicitudes se inician en el receptor, usualmente un navegador web.

Los clientes y los servidores se comunican intercambiando mensajes individuales. Estos mensajes son enviados por el cliente y se llaman solicitudes (requests), y los mensajes enviados por el servidor son llamados respuestas (responses).

Cuando clickeas en un link en una pagina web, envias un formulario o haces una busqueda, el navegador envia una peticion HTTP a un servidor.

Esta peticion incluye:

  • Una URL que identifica el servidor y el recurso (por ejemplo: un archivo HTML)
  • Un metodo que define que accion se requiere. (ejemplo: obtener un archivo, guardar o actualizar datos).

    Algunos de los verbos HTTP posibles son:

    • GET: Para obtener un recurso especifico.
    • POST: Para crear un nuevo recurso.
    • PUT: Actualizar un recurso existente
    • DELETE: Eliminar un recurso especifico.

La peticion HTTP tambien puede incluir informacion adicional codificada en la URL (query string), datos asociados a un verbo POST y datos asociados a cookies.

Los servidores web responden a las peticiones de los clientes, las procesan y les dan una respuesta HTTP. Esta respuesta contiene un estado (status) indicando si la solicitud fue exitosa o no. (Por ejemplo, el codigo de estado 200 OK , indica que todo salio bien, mientras que un estado 404 Not found, indica que el recurso no pudo ser encontrado).

El cuerpo (body) de una respuesta exitosa contiene el recurso solicitado. (ej: una pagina HTML, o una imagen), la cual sera mostrada en el navegador.

2. Tipos de sitio web

2.1. Sitios estaticos

Un sitio estatico es aquel que siempre devuelve el mismo contenido "hard-codeado" desde el servidor cuando se solicita un mismo recurso (Fig. 1). Cuando un usuario navega a una pagina, el navegador envia una solicitud HTTP GET a una URL especifica.

El servidor retorna el documento solicitado de su sistema de archivos, junto con una respuesta HTTP que contiene el documento y su respectivo status HTTP.

2.2. Sitios dinamicos

En un sitio dinamico el contenido de la respuesta se genera de forma dinamica (valga la redundancia). Las paginas HTML normalmente se crean insertando datos desde una base de datos dentro de plantillas HTML (templates). Esta es una forma mucho mas eficiente de guardar una gran cantidad de contenido que usando sitios estaticos.

Un sitio dinamico puede retornar distintos datos para una URL basada en la informacion del usuario y puede realizar otras operaciones como parte de la respuesta (por ejemplo: enviar notificaciones).

El esquema a continuacion muestra la interaccion tipica de un sitio dinamico. Las solicitudes del navegador (1) son interpretadas por el servidor (2), se obtiene la informacion requerida en la base de datos (3) y se combina con plantillas HTML (4) para luego enviar la respuesta que contiene el HTML generado (5, 6).

3. Archivos en una aplicacion web

Un sitio web consiste de distintos archivos: texto, codigo, hojas de estilo (css), contenido multimedia (imagenes, videos), entre otras.

Para crear un sitio es necesario que estos archivos se estructuren de forma que puedan comunicarse entre si. Cuando creamos una pagina web en nuestra computadora local, debemos mantener todos los archivos en un mismo directorio.

3.1. Estructura del sitio

Los archivos mas comunes que tendremos en cualquier proyecto de pagina web son un archivo index HTML (.html), carpetas que contengan imagenes (.png, .jpg), archivos CSS (.css) y archivos Javascript (.js).

Creemos una pagina simple de ejemplo para poder ver como interactuan cada uno de estos:

Vamos a crear un archivo index.html basico para empezar a familiarizarnos con esta estructura. 

<!DOCTYPE html>
<html lang="es">
    <head>
        <title>Mi primer pagina</title>
        <meta charset="UTF-8">
        <meta name="viewport" content="width=device-width, initial-scale=1">
        <link href="style.css" rel="stylesheet">
    </head>
    <body>
        Hello world!
    </body>
</html>

Vamos a hacer la prueba de crear este archivo con algun editor de texto. Nosotros recomendamos Visual Studio Code, pero pueden utilizar cualquiera de su preferencia.

  1. Instalar Visual Studio Code
  2. Crear una carpeta llamada hello-world
  3. Abrir la carpeta creada en Visual Studio Code
  4. Crear un archivo llamado index.html y pegar el codigo html mostrado anteriormente.
  5. Abrir el archivo index.html con un navegador web (eg: Google Chrome, Mozilla Firefox)

3.2. Archivos HTML, CSS y Javascript

Estos 3 archivos son los pilares fundamentales de todo sitio web.

HTML (HyperText Markup Language) es el codigo que se utiliza para estructurar la pagina web y su contenido. Por ejemplo, el contenido puede estar estructurado en un conjunto de parrafos, una lista de items, imagenes, tablas.

CSS (Cascading Style Sheets) es el codigo que le da el estilo al sitio web. Algunas de las cosas que podemos hacer con CSS, pueden ser: cambiar colores del fondo o texto, alinear el contenido, cambiar tama√±o de imagenes, etc.

Javascript es un lenguaje de programacion que permite agregar interactividad a una pagina web. Por ejemplo, en el comportamiento de un boton al ser clickeado, o el ingreso de datos en un formulario. Tambien con estilos dinamicos, por ejemplo cambiar el color de un enlace cuando ponemos el mouse encima, o una animacion.
