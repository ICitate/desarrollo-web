1. Navegadores web

Un navegador web es una aplicacion que sirve para acceder a sitios web. Cuando un usuario ingresa a una pagina web, el navegador trae los archivos de esta pagina desde un servidor web, y los muestra en pantalla. Los navegadores web pueden ser utilizados en diferentes dispositivos como computadoras, celulares y tablets.

Para que todos los navegadores web interpreten el contenido de la misma manera, se crearon los "estandares web" (web standards). Son una serie de especificaciones tecnicas, que detallan como deberian funcionar las tecnologias web. Uno de los aspectos clave de los estandares web es que son abiertos para su uso y contribucion. Por lo tanto, cualquier persona puede programar un sitio web de forma gratuita. Esto tambien significa que ninguna empresa puede tomar el control de estos estandares.

Los hipervinculos (links) son los que permiten a los usuarios navegar hacia distintas paginas al hacer click sobre ellos.

El hipertexto es texto que contiene hipervinculos, y HTML (Lenguaje de marcado de hipertexto) nos permite describir una pagina web que contiene hipertexto.

Cada pagina web tiene una URL (Uniform Resource Locator) unica, tambien conocida como direccion web, que especifica la ubicacion de la misma en una red de computadoras. Por ejemplo: https://www.google.com es la URL que apunta a la pagina principal de el buscador de Google.

Los sitios web guardan informacion en archivos llamados cookies 游꼵. Estos se guardan en tu computadora para la proxima vez que visites el sitio. Por ejemplo, cuando una pagina web recuerda tu usuario y contrase침a esta utilizando una cookie.

1.1 Funcionalidad de un navegador

La funcion principal de un navegador es el de presentar el recurso web elegido al usuario, mediante una solicitud al servidor web correspondiente.

Este recurso es usualmente un documento HTML, pero tambien puede ser un archivo PDF, una imagen, un video u otro tipo de contenido.

La forma en la que el navegador interpreta y muestra los archivos HTML y CSS se encuentra especificado en las especificaciones de la W3C (World Wide Web Consortium).

1.2 Componentes de un navegador

Los componentes principales de un navegador son:
  1. Interfaz de usuario: esto incluye la barra de direcciones, los botones de atras y adelante, el menu de favoritos, etc.
  2. Motor del navegador: Se encarga de comunicar las acciones entre la interfaz de usuario y el motor de renderizado.
  3. Motor de renderizado: Se encarga de mostrar el contenido solicitado. Transformar los documentos HTML, CSS y otros recursos de una pagina web en una representacion visual interactiva en el dispositivo del usuario.
  4. Redes: Se encarga de realizar solicitudes de red (Internet) como las solicitudes HTTP, utilizando diferentes implementaciones para diferentes plataformas detras de una interfaz independiente de la plataforma en cuestion.
  5. UI Backend: Se utiliza para dibujar los elementos basicos como ventanas, botones e inputs de usuario. Este backend expone una interfaz generica que no es especifica a la plataforma. Detras de escenas utiliza la interfaz de usuario del sistema operativo.
  6. Interprete Javascript: Se utiliza para parsear y ejecutar codigo Javascript.
  7. Almacenamiento de datos: Es una capa de persistencia. El navegador necesita guardar datos de forma local, como las cookies. Tambien existen otros tipos de almacenamiento como localStorage, IndexedDB, WebSQL y FileSystem.

Los navegadores como Google Chrome suelen utilizar varias instancias del motor de renderizado, uno para cada pesta침a. Cada pesta침a se ejecuta en un proceso diferente.

2. Concepto de Multiplataforma

El termino plataforma se puede referir a un sistema operativo o un hardware especifico. En el caso de la web, la plataforma es el navegador web. Una pagina, programa o software multiplataforma es aquella que puede funcionar en distintas plataformas sin necesidad de cambiar el codigo para que funcione en alguna en especifico.

Las paginas y aplicaciones web son multiplataforma dado que funcionan en distintos navegadores (Google Chrome, Mozilla Firefox, Opera) y dispositivos (smartphone, escritorio, tablet, smart TV).

Esta es una gran ventaja, a favor de las aplicaciones web, ya que permiten que el desarrollador mantenga un mismo codigo, para varias plataformas. Por ejemplo, en el caso de las aplicaciones nativas, como Spotify, usualmente poseen diferentes versiones de la misma aplicacion para ser compatible con todos los dispositivos (Android, iOS, Windows, Mac, Linux, etc).
